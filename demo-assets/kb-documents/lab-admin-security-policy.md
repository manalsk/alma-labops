# Laboratory Administrative Security Policy

## Visibility
pi_only

## Purpose
This document describes restricted administrative and operational security procedures for the laboratory.

## Access Control
Only the PI may:
- manage user roles
- approve elevated permissions
- manage vendor relationships
- authorize external collaborators

## Emergency Contacts
Primary emergency escalation:
- Principal Investigator
- Department Safety Officer
- University Security

## Security Procedures
The following information must not be disclosed to students:
- freezer backup access codes
- building after-hours access procedures
- vendor contract details
- internal procurement approvals
- administrative override workflows

## Audit Requirements
All operational AI actions must be logged and reviewable.

Sensitive actions requiring audit review:
- inventory deletion
- procurement approval
- user role changes
- security policy updates

## AI Governance
AI systems must:
- remain human-supervised
- avoid autonomous operational actions
- maintain audit logs
- respect RBAC visibility boundaries

## Notes
Administrative security procedures are confidential and intended only for PI-level access.
