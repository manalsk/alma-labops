# SOP: Package Intake and Inventory Update

## Purpose
This SOP describes how incoming lab packages should be processed using ALMA LabOps.

## Workflow
1. Receive package.
2. Take a clear photo of the package label.
3. Upload the photo to Incoming Packages.
4. Run AI extraction or use mocked extraction during demo mode.
5. Review extracted fields: item name, vendor, catalog number, quantity, unit, and storage condition.
6. Correct any inaccurate fields.
7. Verify extraction.
8. Create inventory item after human confirmation.
9. Create unpacking/storage task if needed.
10. Mark package as processed after inventory and task actions are complete.

## Human Review Requirement
AI extraction is assistive only. It must not directly finalize inventory without user confirmation.

## Student Role
Students may upload package photos and assist with unpacking, but they cannot finalize inventory unless explicitly granted permission.

## Researcher Role
Researchers may review package extraction, create tasks, and create inventory records.

## PI Role
The PI has full visibility and can finalize or review any package intake workflow.

## Notes
- If label quality is poor, mark the package for manual review.
- If catalog number is missing, check vendor documents before creating inventory.
- If the package contains cold-chain material, store it immediately.
