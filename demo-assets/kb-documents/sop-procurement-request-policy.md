# SOP: Procurement Request Policy

## Purpose
This SOP explains how lab members should request new items while reducing duplicate orders and unnecessary spending.

## Request Rules
1. Search inventory before creating a purchase request.
2. Check whether an existing pending request already exists.
3. Include item name, vendor, catalog number, quantity, reason for request, and urgency.
4. Link the request to a low-stock inventory item where possible.
5. The PI must approve purchases before ordering.

## Approval Authority
Only the PI can approve purchase requests because the PI manages the grant budget.

## Researcher Permissions
Researchers can create purchase requests and update their own pending requests.

## Student Permissions
Students can create purchase suggestions. These must be reviewed by a researcher or PI before becoming official purchase requests.

## Duplicate Prevention
If a requested item is already in stock or already pending approval, ALMA LabOps should flag the request.

## Urgency Levels
- Low: useful but not immediately needed
- Medium: needed soon
- High: needed for active experiment
- Urgent: experiment blocked without item

## Notes
Expensive or unusual items should include justification and preferred vendor information.
