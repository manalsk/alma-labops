# ALMA LabOps — Setup Instructions

---

## Prerequisites

- Node.js 20+ and npm
- Python 3.12+
- [uv](https://docs.astral.sh/uv/) (Python package manager) — install with `curl -LsSf https://astral.sh/uv/install.sh | sh`
- A Supabase project (free tier is sufficient)
- An OpenAI API key (GPT-4o and gpt-4.1-mini access required)
- Git

---

## 1. Clone the Repository

```bash
git clone <repo-url>
cd alma-labops
```

---

## 2. Supabase Project Setup

### 2.1 Create a Supabase project

Go to [supabase.com](https://supabase.com), create a new project. Take note of:

- Project URL (e.g. `https://abcxyz.supabase.co`)
- Anon public key
- Service role key (keep this secret — backend only)

### 2.2 Enable the pgvector extension

In the Supabase dashboard → SQL Editor, run:

```sql
CREATE EXTENSION IF NOT EXISTS vector;
```

### 2.3 Run migrations

Apply all migrations in order using the SQL Editor. Paste and run each file:

```
supabase/migrations/001_schema.sql       — core tables, RLS, audit logs
supabase/migrations/002_inventory.sql    — inventory items, locations, categories
supabase/migrations/003_procurement.sql  — vendors, purchase requests, items
supabase/migrations/004_tasks.sql        — tasks, task activity logs
supabase/migrations/005_incoming_packages.sql  — packages, ai_audit_logs extensions
supabase/migrations/006_knowledge_base.sql     — KB documents, chunks, RAG queries, pgvector index
supabase/migrations/007_copilot.sql            — copilot_queries table
supabase/migrations/008_permissions_catalog.sql — delegatable permissions catalog
```

Run them in the order listed. Each migration is idempotent (`IF NOT EXISTS` / `ON CONFLICT DO NOTHING`).

### 2.4 Create storage buckets

In the Supabase dashboard → Storage, create two **private** buckets:

| Bucket name | Purpose | Recommended max size |
|---|---|---|
| `package-images` | Incoming package photos | 10 MB per file |
| `kb-documents` | KB SOPs and policy docs | 50 MB per file |

Both buckets must be **private** (no public access). All file access goes through signed URLs generated by the backend.

## 3. Backend Setup

### 3.1 Create the environment file

```bash
cd backend
cp .env.example .env
```

Edit `backend/.env`:

```env
# OpenAI — backend only, never expose to frontend
OPENAI_API_KEY=sk-...

# Supabase — service role key is backend only
SUPABASE_URL=https://your-project.supabase.co
SUPABASE_SERVICE_ROLE_KEY=eyJ...

# CORS — set to your frontend URL
CORS_ORIGINS=http://localhost:3000
```

> **Important:** The `SUPABASE_SERVICE_ROLE_KEY` bypasses Row Level Security. It must never appear in any `NEXT_PUBLIC_*` variable or be committed to version control.

### 3.2 Install dependencies

```bash
cd backend
uv sync
```

This creates a `.venv` and installs all dependencies from `pyproject.toml`.

### 3.3 Run the backend

```bash
cd backend
uv run uvicorn app.main:app --reload --port 8000
```

Verify it's running:
- Health check: [http://localhost:8000/health](http://localhost:8000/health)
- API docs: [http://localhost:8000/docs](http://localhost:8000/docs)

## 4. Frontend Setup

### 4.1 Create the environment file

```bash
cd frontend
cp .env.example .env.local
```

Edit `frontend/.env.local`:

```env
# Supabase — anon key is safe for the browser
NEXT_PUBLIC_SUPABASE_URL=https://your-project.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=eyJ...

# Backend API URL
NEXT_PUBLIC_API_URL=http://localhost:8000
```

### 4.2 Install dependencies

```bash
cd frontend
npm install
```

### 4.3 Run the frontend

```bash
cd frontend
npm run dev
```

Frontend is available at [http://localhost:3000](http://localhost:3000).

## 5. Seed Demo Data

All seed scripts are run from the `backend/` directory. Run them in the order listed — users must exist before inventory, packages, etc.

```bash
cd backend

# 1. Create demo user accounts in Supabase Auth + profiles
uv run python -m scripts.seed_users

# 2. Seed inventory locations, categories, and items
uv run python -m scripts.seed_inventory

# 3. Seed vendors and purchase requests
uv run python -m scripts.seed_procurement

# 4. Seed tasks
uv run python -m scripts.seed_tasks

# 5. Seed incoming packages (requires images already in package-images/demo/ bucket)
uv run python -m scripts.seed_packages

# 6. Seed knowledge base documents (requires files already in kb-documents/demo/ bucket)
uv run python -m scripts.seed_knowledge_base
```

> **Note for seeds 5 and 6:** Demo image and document files must be uploaded to Supabase Storage before running those seed scripts. Upload the contents of `demo-assets/package-images/` to the `package-images/demo/` path in your bucket, and `demo-assets/kb-documents/` to `kb-documents/demo/`.

## 7. Knowledge Base: Upload Documents

The seed script expects documents already uploaded to Supabase Storage. To manually upload:

1. Go to Supabase dashboard → Storage → `kb-documents` bucket
2. Create a folder called `demo/`
3. Upload each `.md` file from `demo-assets/kb-documents/` into `kb-documents/demo/`
4. Then run: `uv run python -m scripts.seed_knowledge_base`

This will:
- Create `kb_documents` rows for each document
- Chunk and embed each document using `text-embedding-3-small`
- Store chunk vectors in `document_chunks`
- Mark each document as indexed with a chunk count

To add new documents after setup: use the Upload button on the `/knowledge-base` page (PI or researcher with `upload_kb_docs` permission).

## 8. Running AI Evals

```bash
cd backend

# Run all eval cases (RAG + Copilot)
uv run python -m scripts.run_basic_ai_evals

# Run only RAG eval cases
uv run python -m scripts.run_basic_ai_evals --system rag

# Run only Copilot eval cases
uv run python -m scripts.run_basic_ai_evals --system copilot

# Run a single case by ID
uv run python -m scripts.run_basic_ai_evals --id cop_04
```

Requires `OPENAI_API_KEY` in `backend/.env`. Exit code 0 = all pass; exit code 1 = at least one failure. Eval cases are defined in `evals/basic_ai_eval_cases.json`.

## 9. Troubleshooting

**Backend fails to start with `ImportError`**
Run `uv sync` to ensure all dependencies are installed.

**`postgrest.exceptions.APIError: relation "..." does not exist`**
A migration was not applied. Check the Supabase SQL Editor and run the missing migration.

**`APIError: foreign key constraint violation on user_permissions`**
Migration `008_permissions_catalog.sql` was not applied. Run it in the SQL Editor.

**`seed_knowledge_base.py` reports missing file**
Documents must be uploaded to Supabase Storage at `kb-documents/demo/` before running the seed. See Section 7.

**Frontend shows "Not authenticated" or redirects loop**
Check that `NEXT_PUBLIC_SUPABASE_URL` and `NEXT_PUBLIC_SUPABASE_ANON_KEY` in `frontend/.env.local` match your Supabase project exactly.

**Copilot returns 502**
The backend's OpenAI call failed. Check that `OPENAI_API_KEY` is set in `backend/.env` and is valid. The backend logs the full traceback at the `ERROR` level.

**CORS errors in browser console**
Ensure `CORS_ORIGINS` in `backend/.env` includes your frontend URL (e.g. `http://localhost:3000`).
